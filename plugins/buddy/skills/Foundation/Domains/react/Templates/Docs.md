# Documentation Template: [PROJECT NAME]

## Required Documentation Files

### architecture.md
- Application architecture overview
- Component hierarchy diagram (mermaid)
- State management architecture
- Data flow patterns
- Rendering strategy (CSR/SSR/SSG)
- Technology stack summary

### api-reference.md *(if applicable)*
- API integration layer
- Custom hooks API
- Context providers API
- Component props API

### setup.md
- Prerequisites (Node version, package manager)
- Installation steps
- Environment variables
- Running development server
- Running tests
- Building for production

### component-guide.md *(if design system/shared components)*
- Component catalog
- Props documentation
- Usage examples
- Storybook reference (if applicable)

### deployment.md
- Build process
- Environment configuration
- Hosting setup (Vercel, Netlify, AWS, etc.)
- CI/CD pipeline
- Performance monitoring

### troubleshooting.md
- Common development issues
- Build troubleshooting
- State management debugging
- Performance profiling
- Browser compatibility issues

## Documentation Guidelines
- Include mermaid diagrams for component hierarchies and data flow
- Show real component usage examples from the codebase
- Document custom hooks with input/output types
- Include accessibility guidelines specific to the project
